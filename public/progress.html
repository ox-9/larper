🐛 Bugs & Issues

HIGH

Invalid CSS: rgba: instead of rgba(

NewfiBaselinePanel and GuidelineSkeleton use broken Tailwind syntax like bg-[rgba:255,255,255,0.08)] — these styles silently fail to apply.

HIGH

TypeScript type mismatch: activePage missing "chat"

lib/types.ts defines activePage: "upload" | "compare" but the app actually uses a 3-tab layout including "chat". Runtime works, type system is wrong.

MED

Duplicate inline ComparisonView in page.tsx

page.tsx defines its own ComparisonView, ScoreRing, and ComparisonResults inline while ComparisonPage.tsx in components/ has equivalent but different versions — maintenance risk.

MED

Duplicate/conflicting Tailwind dark classes

Multiple files chain conflicting dark:text-slate-500 dark:text-slate-400 — later class wins, earlier one is dead code creating specificity confusion.

MED

Static export incompatible with API routes

next.config.ts uses output: "export" but 7 API route handlers exist under app/api/. These routes won't function in a static deployment — dead code and deployment confusion risk.

LOW

Gemini pageCount heuristic is inaccurate

pageCount: Math.ceil(file.size / 50000) estimates pages from file size, which is unreliable — embedded fonts and images skew it significantly.

LOW

Potential chat abort controller leak

If LarpGPT component unmounts during streaming, the abort controller in abortRef may not be properly cleaned up.

LOW

Legacy CSS class aliases

Both glass/glass-strong/glass-card AND liquid-glass/liquid-glass-strong/liquid-glass-card serve identical purposes. The liquid- variants are marked "Legacy aliases" but still present.

✨ Polish & Adjustments Needed

1

Delete 10 unused components

ComplianceScore, FeatureCard, FileUpload, HistoryPanel, ProcessingIndicator, Recommendations, StatsCard, TabAnalysis, HeaderReviewModal, ComparisonPage — all dead code from an earlier iteration. Removing them reduces bundle size and confusion.

2

Remove 7 dead API route handlers

app/api/analyze/, ai/, upload/, extract-v3/, extract/, chat/, ollama-extract/ — all unused in static export deployment. They add build time and confusion.

3

Fix rgba: CSS syntax bugs

NewfiBaselinePanel and GuidelineSkeleton have rgba:255,255,255,0.08) instead of rgba(255,255,255,0.08). These silently produce no style.

4

Fix activePage type to include "chat"

Add "chat" to the union type in lib/types.ts so TypeScript matches the actual runtime behavior.

5

Deduplicate ComparisonView / ScoreRing / ComparisonResults

Extract the inline versions from page.tsx into proper components, or consolidate onto the existing ComparisonPage.tsx. Currently two divergent implementations exist.

6

Remove legacy liquid-glass CSS aliases

The liquid-glass / liquid-glass-strong / liquid-glass-card classes in globals.css are marked "Legacy aliases" and duplicate the glass variants. Clean up and update any references.

7

Clean up duplicate/conflicting dark: Tailwind classes

Files like DocumentUploadCard and GuidelineList chain dark:text-slate-500 dark:text-slate-400 — only the last one applies. Remove the dead earlier ones.

8

Add error UI for extraction failures

When both client-side and Gemini extraction fail, show a user-facing toast or inline error message explaining what went wrong and suggesting next steps.

9

Replace mock baseline data

newfiGuidelines.ts has only 10 hardcoded rows marked "Mock." Replace with real NewFI wholesale underwriting guidelines for production use.

10

Wire up HistoryPanel or remove it

history.ts and HistoryPanel.tsx exist but are not connected. Either integrate history into the app flow (save/load analysis sessions) or delete both files.

11

Remove unused npm dependencies

pdf-parse, pdf2json, xlsx, openai, and @anthropic-ai/sdk are in package.json but the static-export app doesn't use them (client-side uses pdfjs-dist and Gemini directly).

12

Improve overlay comparison logic

Current overlay merge does a naive [Overlay Adjustment]: text string append. This should do a proper guideline-level merge with conflict detection and adjustment tracking.
